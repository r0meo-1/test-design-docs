# BUG-003 — IDOR: доступ к чужой заявке по прямому id

| Поле | Значение |
|------|----------|
| ID | BUG-003 |
| Severity | Critical |
| Priority | P1 |
| Компонент | Авторизация / API |
| Окружение | staging, API v2.3 |
| Статус | In Progress |

## Описание
Авторизованный пользователь может получить **чужую** заявку, подставив чужой `id`
в `GET /bookings/{id}` — раскрываются персональные данные другого клиента (IDOR).

## Шаги воспроизведения
1. Войти под пользователем A, узнать формат `GET /bookings/{id}`.
2. Подставить `id` заявки пользователя B.
3. Выполнить запрос с токеном пользователя A.

## Фактический результат
200 OK с данными заявки B: ФИО, телефон, сумма, тур. Проверка владельца отсутствует.

## Ожидаемый результат
Сервер сверяет владельца заявки с текущим пользователем → **403 Forbidden** (или 404),
данные не раскрываются.

## Вложения
- Postman: два запроса (токен A → заявка A: 200; токен A → заявка B: ожидается 403, фактически 200).
- Связанный тест-кейс: `test-cases/auth-and-access.md` → TC-AUTH-03.

## Влияние
Утечка ПДн клиентов, нарушение требований по защите данных. Требует приоритетного фикса.

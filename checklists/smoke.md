# Smoke-чек-лист (после каждого релиза)

Цель: за 10–15 минут убедиться, что ключевые сценарии живы. Любой `[ ]` → не выкатываем дальше / откат.

## Доступность
- [ ] Главная страница открывается, нет 5xx
- [ ] Каталог туров загружается, видны карточки и цены
- [ ] Статические ассеты грузятся (нет битых стилей/картинок в DevTools → Network)

## Бронирование
- [ ] Открывается карточка тура, видно число свободных мест
- [ ] Создаётся бронь, появляется номер заявки
- [ ] Заявка видна в личном кабинете в статусе `new`

## Оплата
- [ ] Открывается форма оплаты с корректной суммой
- [ ] Успешная оплата переводит заявку в `paid`
- [ ] Отказ оплаты не меняет статус на `paid`

## Личный кабинет / доступ
- [ ] Вход с валидными данными работает
- [ ] Чужие заявки по прямому id недоступны

## API / данные
- [ ] Ключевые эндпоинты отвечают 2xx (Postman/Newman smoke)
- [ ] Нет новых нарушений data-quality (SQL-проверки)

# Регрессионный чек-лист

Прогоняется перед крупными релизами и при изменениях в платежах/бронировании.

## Каталог и поиск
- [ ] Поиск по направлению/датам возвращает релевантные туры
- [ ] Фильтры (цена, страна, даты) применяются и сбрасываются
- [ ] Пагинация/подгрузка работает, нет дублей карточек
- [ ] Цена и наличие мест совпадают с карточкой тура

## Бронирование
- [ ] Граничные значения числа туристов (1, max, max+1, 0)
- [ ] Перерасчёт суммы при изменении состава
- [ ] Отмена брони возвращает места
- [ ] Валидация контактных данных (телефон, email)

## Оплата и возвраты
- [ ] Успешная оплата, отказ, таймаут
- [ ] Идемпотентность повторной оплаты (нет двойного списания)
- [ ] Полный и частичный возврат
- [ ] Сверка суммы заявки и платежей

## Авторизация и безопасность
- [ ] Вход/выход, восстановление пароля
- [ ] Защита от перебора пароля
- [ ] IDOR: нет доступа к чужим заявкам
- [ ] Защищённые страницы недоступны без сессии

## Интеграции
- [ ] Платёжный шлюз: статусы и колбэки
- [ ] Поставщики турпродукта: обработка ошибок и таймаутов
- [ ] Уведомления (Telegram-бот / n8n) приходят по событиям

## Данные
- [ ] Нет дублей клиентов
- [ ] Нет «сиротских» заявок/платежей
- [ ] Статусная модель консистентна (paid↔платёж, refunded↔возврат)

# Тест-кейсы — сценарий бронирования

Приоритеты: **P1** — критично, **P2** — важно, **P3** — второстепенно.

## TC-BKG-01 (P1) — успешное бронирование тура

**Предусловие:** пользователь авторизован; тур с наличием мест выбран.

| # | Шаг | Ожидаемый результат |
|---|-----|---------------------|
| 1 | Открыть карточку тура с доступными местами | Отображается цена, даты, число свободных мест |
| 2 | Указать 2 туристов и контактные данные | Поля валидны, итоговая сумма = цена × 2 |
| 3 | Нажать «Забронировать» | Создаётся заявка в статусе `new`, виден номер заявки |
| 4 | Открыть личный кабинет | Заявка отображается в истории со статусом `new` |

**Ожидаемый итог:** заявка создана, сумма рассчитана корректно, данные сохранены в CRM.

## TC-BKG-02 (P1) — бронирование при отсутствии мест

**Предусловие:** у тура `seats_total` исчерпан.

| # | Шаг | Ожидаемый результат |
|---|-----|---------------------|
| 1 | Открыть карточку тура без мест | Кнопка «Забронировать» недоступна / показано «Мест нет» |
| 2 | Попытаться отправить бронь через прямой запрос | Сервер отклоняет: 409/422, заявка не создаётся |

## TC-BKG-03 (P2) — граничное число туристов

| # | Шаг | Ожидаемый результат |
|---|-----|---------------------|
| 1 | Указать 0 туристов | Валидация: значение должно быть ≥ 1 |
| 2 | Указать число больше свободных мест | Валидация: «Недостаточно мест» |
| 3 | Указать максимально доступное число | Бронь проходит |

## TC-BKG-04 (P2) — перерасчёт суммы при изменении параметров

| # | Шаг | Ожидаемый результат |
|---|-----|---------------------|
| 1 | Создать бронь на 2 туристов | Сумма = цена × 2 |
| 2 | Изменить число туристов на 3 до оплаты | Сумма пересчитывается = цена × 3 |

## TC-BKG-05 (P1) — отмена брони

| # | Шаг | Ожидаемый результат |
|---|-----|---------------------|
| 1 | Открыть заявку в статусе `new` | Доступно действие «Отменить» |
| 2 | Подтвердить отмену | Статус → `cancelled`, место возвращается в доступные |

## TC-BKG-06 (P3) — невалидные контактные данные

| # | Шаг | Ожидаемый результат |
|---|-----|---------------------|
| 1 | Ввести некорректный телефон/email | Поле подсвечено, отправка заблокирована |
